古典的なミッション「 Open Labyrinth」を覚えているだろうか? そう…あなたは気が付くとまたしても迷宮の中にいた。 しかし明かりが無く全体地図を見ることもできない。 あなたは懐中電灯を持っており、東西南北の方向に沿った迷宮内の道を全て知ることができる。 だが現在地および出口の位置は分からない。 あなたは永遠に動き続けられる訳ではないので急がなくてはならない。

これは「マルチコール(malticall)」ミッションである。そのため、あなたの関数はこの迷路を解くか 動けなくなるまで繰り返し呼ばれる。 毎回、4方向を見ることができる。(これが関数の入力値となる) あなたは道を見通すことができ、それらについて交差点と曲がり角が分る。 またこの視界内に出口あればそれも識別可能である。 視界はstring型のタプルによるグリッドで表わされる。 "X" は壁、 "." は道、 "?" は不明、 "P" は自分、 "E" は出口 である。 グリッドは2次元の長方形であり、これの大きさは視界に依る。

毎回、あなたの関数は1つか複数の行動をstring型の方向を示す文字列で返すこと。 行動は'N' - 北, 'S' - 南, 'W' - 西, 'E' - 東で記述する。 例えば 'NWS'などで一連の移動を表現する。 壁は奇妙なスライムで覆われているので壁に向かって移動してはならない。 興味深いことにあなたは最大250歩しか歩けない。(これは関数が呼ばれる回数ではない) ミッションを通じてグローバル変数を使用してもよい。

以下があなたの迷宮内における視界の例である。 これが関数の入力値となる: 

("???XXX",
 "???...",
 "???X.X",
 "XXXX.X",
 "X...PX",
 "XXXX.X",
 "???X.X",
 "???...",
 "???X.X",
 "???X.X",
 "???..X",
 "???XXX")

入力: 視界。(string型のタプル)

出力: 1回もしくは複数回の行動(string型)

例:

どのように使われるか: これは経路探索アルゴリズムとその探索がコンセプトである。 領域内を探索し地図を作製するのに利用できる。

事前条件:
迷宮は壁に囲まれている。
全ての道は狭い。(幅1マス)
3 < len(visible) ≤ 15
all(len(row) == len(visible[0]) for row in visible)
